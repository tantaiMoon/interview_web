# 算法面试题

## 一、算法基础认知

### 说说你对算法的理解？应用场景是什么？

算法本质上是“解决某一类问题的步骤和策略”。

在面试语境里，算法不是单纯写题，而是考察你：

1. 如何抽象问题。
2. 如何选择合适的数据结构。
3. 如何控制时间和空间复杂度。
4. 如何把通用解法迁移到实际业务场景。

常见应用场景：

1. 搜索和排序。
2. 路径规划和推荐。
3. 缓存淘汰策略。
4. 文本匹配和数据去重。
5. 调度、分页、合并、统计等工程问题。

一句话总结：算法是“解决问题的方法”，数据结构是“组织数据的方式”，两者通常配合出现。

### 说说你对算法中时间复杂度、空间复杂度的理解？如何计算？

时间复杂度用来描述算法执行时间随输入规模增长的趋势，空间复杂度用来描述算法额外占用空间随输入规模增长的趋势。

常见复杂度从低到高大致是：

1. `O(1)`
2. `O(log n)`
3. `O(n)`
4. `O(n log n)`
5. `O(n^2)`
6. `O(2^n)`
7. `O(n!)`

计算时通常看：

1. 循环嵌套层数。
2. 递归树展开规模。
3. 是否使用额外数组、哈希表、递归栈。

面试时要强调：复杂度关注的是趋势和数量级，而不是某次具体运行时间。

### 说说你对数据结构的理解？有哪些？区别是什么？

数据结构是组织和存储数据的方式，决定了数据的访问、插入、删除和查找效率。

常见数据结构有：

1. 数组
2. 链表
3. 栈
4. 队列
5. 哈希表
6. 树
7. 堆
8. 图
9. 集合
10. 并查集

它们的区别主要体现在：

1. 存储方式不同。
2. 访问方式不同。
3. 查找、插入、删除成本不同。
4. 适合解决的问题类型不同。

### 算法刷题前应该先建立哪些基础？

刷题前最先建立的是“题型识别能力”，而不是背答案。

建议先掌握：

1. 复杂度分析。
2. 常见数据结构操作。
3. 递归与迭代转换。
4. 模板化思维，比如二分、双指针、滑动窗口、DFS/BFS。
5. 边界条件意识，比如空数组、单节点、越界、重复元素。

代表题型：

1. 复杂度判断题。
2. 数组遍历与原地修改。
3. 递归树基础。
4. 简单查找、插入、删除模拟题。

## 二、线性结构专题

### 说说你对数组的理解？常见的算法题思路有哪些？

数组是连续内存空间的数据结构，随机访问效率高，但中间插入和删除成本较高。

数组类题目的高频思路有：

1. 遍历模拟。
2. 双指针。
3. 滑动窗口。
4. 前缀和。
5. 二分查找。
6. 原地交换。

面试时可以补一句：数组题最常考的是“有序、连续、下标、区间”这些特征能否被利用起来。

代表题型：

1. 两数之和。
2. 移除元素。
3. 合并区间。
4. 旋转数组。
5. 最大子数组和。
6. 缺失的第一个正数。

### 说说你对字符串的理解？常见的算法题思路有哪些？

字符串可以理解为字符数组，因此很多字符串题本质上还是数组题。

常见思路有：

1. 双指针。
2. 滑动窗口。
3. 哈希计数。
4. 子串匹配。
5. 回文判断。
6. 模拟和状态机。

字符串题的关键通常是：

1. 明确子串、子序列、前缀、后缀的区别。
2. 注意大小写、空格、编码边界和下标处理。

代表题型：

1. 最长公共前缀。
2. 最长无重复子串。
3. 有效字母异位词。
4. 最长回文子串。
5. 字符串相加。
6. 字符串解码。

### 说说你对链表的理解？常见的操作和题型有哪些？

链表是通过指针把节点串联起来的线性结构，不要求连续内存。

优点：插入删除方便。
缺点：随机访问能力差。

链表高频题型：

1. 反转链表。
2. 快慢指针找中点或环。
3. 合并两个有序链表。
4. 删除倒数第 N 个节点。
5. 判断链表是否有环。
6. 链表分割、重排、相交判断。

面试时经常考的不是 API，而是指针操作是否稳定、边界是否处理干净。

代表题型：

1. 反转链表。
2. 合并两个有序链表。
3. 环形链表。
4. 删除链表倒数第 N 个节点。
5. 两两交换链表节点。
6. K 个一组翻转链表。

### 说说你对栈、队列的理解？应用场景是什么？

栈是后进先出，队列是先进先出。

栈常见应用场景：

1. 括号匹配。
2. 单调栈。
3. 函数调用栈。
4. 表达式求值。

队列常见应用场景：

1. BFS 广度优先遍历。
2. 任务调度。
3. 消息缓冲。
4. 单调队列。

一句话总结：栈更适合处理“最近相关”，队列更适合处理“先来先服务”。

代表题型：

1. 有效括号。
2. 最小栈。
3. 用栈实现队列。
4. 用队列实现栈。
5. 单调栈求下一个更大元素。
6. 滑动窗口最大值。

### 说说你对集合和哈希表的理解？常见的操作和应用场景有哪些？

集合和哈希表本质上都依赖哈希思想来实现高效查找。

1. `Set` 更关注“元素是否存在”。
2. `Map` / 哈希表更关注“键值映射”。

常见操作：

1. 插入
2. 删除
3. 查找
4. 去重
5. 计数

常见应用场景：

1. 两数之和。
2. 频次统计。
3. 判断重复元素。
4. 缓存设计。
5. 字符串异位词判断。

哈希类题的核心是“用空间换时间”。

代表题型：

1. 两数之和。
2. 存在重复元素。
3. 字母异位词分组。
4. 最长连续序列。
5. LRU 缓存。
6. 复制带随机指针的链表。

## 三、树、堆与图

### 说说你对树的理解？相关的操作和题型有哪些？

树是一种层次结构数据模型，最常考的是二叉树。

树的高频操作：

1. 前序、中序、后序遍历。
2. 层序遍历。
3. 求高度、深度、节点数。
4. 判断平衡、对称、是否为搜索树。
5. 最近公共祖先。

树题的常见思路：

1. 递归。
2. 迭代模拟栈。
3. BFS 队列遍历。

代表题型：

1. 二叉树前中后序遍历。
2. 二叉树层序遍历。
3. 对称二叉树。
4. 验证二叉搜索树。
5. 二叉树最近公共祖先。
6. 路径总和。

### 说说你对堆的理解？如何实现？应用场景是什么？

堆是一种特殊的完全二叉树，常见分为大顶堆和小顶堆。

特点：

1. 堆顶元素最大或最小。
2. 常用于快速获取最值。

常见应用场景：

1. Top K 问题。
2. 优先队列。
3. 合并有序数据流。
4. 定时任务调度。
5. 堆排序。

实现上通常基于数组，通过下标关系维护父子节点。

代表题型：

1. 数组中的第 K 个最大元素。
2. 前 K 个高频元素。
3. 合并 K 个有序链表。
4. 数据流中的中位数。
5. 最小堆维护 Top K。

### 说说你对图的理解？相关操作和题型有哪些？

图用来描述点和点之间的关系，比树更通用。

常见表示方式：

1. 邻接表。
2. 邻接矩阵。

高频操作和题型：

1. DFS 深度优先遍历。
2. BFS 广度优先遍历。
3. 最短路径。
4. 拓扑排序。
5. 联通分量。
6. 并查集判连通。

图题常见于依赖关系、路径搜索、社交网络、课程安排等问题。

代表题型：

1. 岛屿数量。
2. 课程表。
3. 克隆图。
4. 腐烂的橘子。
5. 单词接龙。
6. 最短路径基础题。

### 说说你对并查集的理解？应用场景是什么？

并查集是一种用于处理“元素是否属于同一集合”的数据结构。

核心操作：

1. `find`：查找根节点。
2. `union`：合并两个集合。

常见优化：

1. 路径压缩。
2. 按秩合并。

应用场景：

1. 判断连通性。
2. 岛屿数量问题。
3. 图中环检测。
4. 动态合并集合问题。

代表题型：

1. 冗余连接。
2. 省份数量。
3. 岛屿数量扩展题。
4. 账号合并。

## 四、查找与排序专题

### 说说常见的排序算法有哪些？区别是什么？

常见排序算法包括：

1. 冒泡排序。
2. 选择排序。
3. 插入排序。
4. 归并排序。
5. 快速排序。
6. 堆排序。

比较时通常从这些维度说：

1. 时间复杂度。
2. 空间复杂度。
3. 是否稳定。
4. 是否适合大规模数据。
5. 是否适合近乎有序的数据。

代表题型：

1. 数组排序。
2. 颜色分类。
3. 合并区间。
4. 会议室安排。
5. 归并排序统计逆序对。

### 说说你对冒泡排序的理解？如何实现？应用场景是什么？

冒泡排序通过相邻元素两两比较，把较大元素逐步“冒泡”到后面。

特点：

1. 思路简单。
2. 平均复杂度 `O(n^2)`。
3. 稳定排序。

它更适合教学和小规模数据，不适合实际大规模场景。

代表题型：

1. 基础排序模板题。
2. 近乎有序数组的教学示例。

### 说说你对选择排序的理解？如何实现？应用场景是什么？

选择排序每一轮从未排序区间中选出最小值，放到已排序区间末尾。

特点：

1. 实现简单。
2. 平均复杂度 `O(n^2)`。
3. 一般不是稳定排序。

它同样更适合教学，不适合大数据量场景。

代表题型：

1. 基础排序模板题。
2. 交换次数受限的教学场景。

### 说说你对插入排序的理解？如何实现？应用场景是什么？

插入排序的思路是把当前元素插入到前面已排序序列中的正确位置。

特点：

1. 平均复杂度 `O(n^2)`。
2. 稳定排序。
3. 对近乎有序的数据表现较好。

实际工程里，小数组排序或某些优化场景会用到插入排序作为基础策略。

代表题型：

1. 小规模数组排序。
2. 近乎有序数组排序。

### 说说你对归并排序的理解？如何实现？应用场景是什么？

归并排序采用分治思想：

1. 先把数组不断拆分。
2. 再把两个有序子数组合并。

特点：

1. 时间复杂度稳定在 `O(n log n)`。
2. 稳定排序。
3. 需要额外空间。

适合场景：

1. 对稳定性有要求。
2. 链表排序。
3. 外部排序。

代表题型：

1. 排序链表。
2. 合并两个有序数组/链表。
3. 逆序对统计。

### 说说你对快速排序的理解？如何实现？应用场景是什么？

快速排序的核心思想是：

1. 选一个基准值。
2. 把数组分成小于基准和大于基准两部分。
3. 递归处理左右区间。

特点：

1. 平均复杂度 `O(n log n)`。
2. 原地排序能力较强。
3. 最坏情况会退化到 `O(n^2)`。

它是实际工程里非常常见的高效排序思想。

代表题型：

1. 快速排序模板。
2. 数组第 K 大元素的快速选择。
3. 颜色分类的原地分区思想。

### 说说你对二分查找的理解？如何实现？应用场景是什么？

二分查找适用于“有序区间查找”。

核心思想是每次比较中间值，把问题规模缩小一半。

特点：

1. 时间复杂度 `O(log n)`。
2. 要求数据有序。

常见应用场景：

1. 查找目标值。
2. 查找左边界、右边界。
3. 答案区间二分。
4. 旋转数组、峰值、最小值类题目。

代表题型：

1. 二分查找。
2. 搜索插入位置。
3. 在排序数组中查找元素的第一个和最后一个位置。
4. 搜索旋转排序数组。
5. 寻找峰值。

## 五、常见算法思想

### 说说你对双指针的理解？应用场景是什么？

双指针是一类非常高频的算法技巧，通常通过两个指针在数组、链表、字符串上协同移动来优化问题。

常见场景：

1. 有序数组求和。
2. 去重。
3. 合并区间。
4. 回文判断。
5. 快慢指针处理链表问题。

它的核心价值是把原本 `O(n^2)` 的部分问题优化到 `O(n)`。

代表题型：

1. 两数之和 II。
2. 三数之和。
3. 盛最多水的容器。
4. 移动零。
5. 判断回文串。

### 说说你对滑动窗口的理解？应用场景是什么？

滑动窗口本质上是维护一个动态区间，随着左右边界移动，实时更新窗口内状态。

常见场景：

1. 最长不重复子串。
2. 最小覆盖子串。
3. 固定长度子数组统计。
4. 连续区间和问题。

滑动窗口的关键是：

1. 窗口何时扩张。
2. 何时收缩。
3. 窗口内状态如何维护。

代表题型：

1. 长度最小的子数组。
2. 最长无重复子串。
3. 最小覆盖子串。
4. 找到字符串中所有字母异位词。

### 说说你对前缀和的理解？应用场景是什么？

前缀和用于快速计算区间和。

核心思想：

1. 预处理出从起点到当前位置的累积和。
2. 任意区间和可用两个前缀和相减得到。

常见场景：

1. 子数组和。
2. 矩阵区域和。
3. 统计区间类问题。

它的本质是“预处理换取快速查询”。

代表题型：

1. 区域和检索。
2. 和为 K 的子数组。
3. 乘积小于 K 的子数组变体。

### 说说你对分而治之的理解？应用场景是什么？

分治思想指的是把大问题拆成若干规模更小、结构相同的子问题，分别解决后再合并结果。

经典应用：

1. 归并排序。
2. 快速排序。
3. 二叉树递归问题。
4. 大整数运算拆分。

面试时可以强调：分治的关键不只是“拆”，还包括“子问题独立”和“结果合并”。

代表题型：

1. 归并排序。
2. 快速排序。
3. 不同二叉树分治递归题。

### 说说你对贪心算法的理解？应用场景是什么？

贪心算法是在每一步都做当前看起来最优的选择，希望最终得到全局最优解。

适用前提通常是：

1. 问题具有贪心选择性质。
2. 局部最优能导向全局最优。

常见场景：

1. 区间调度。
2. 零钱问题的特定版本。
3. 跳跃游戏。
4. 最少覆盖区间。

代表题型：

1. 跳跃游戏。
2. 分发饼干。
3. 无重叠区间。
4. 买卖股票的最佳时机 II。

### 说说你对回溯算法的理解？应用场景是什么？

回溯本质上是一种“试错 + 撤销”的搜索方法，通常用来枚举所有可能解或寻找满足条件的解。

常见场景：

1. 全排列。
2. 子集。
3. 组合问题。
4. N 皇后。
5. 路径搜索。

回溯题的关键是：

1. 路径是什么。
2. 选择列表是什么。
3. 结束条件是什么。
4. 如何剪枝。

代表题型：

1. 全排列。
2. 子集。
3. 组合总和。
4. N 皇后。
5. 单词搜索。

### 说说你对深度优先搜索和广度优先搜索的理解？区别是什么？

DFS 和 BFS 都是图和树的基础搜索策略。

1. DFS 倾向“一条路走到底再回头”，常用递归或栈实现。
2. BFS 倾向“按层扩展”，常用队列实现。

区别主要在：

1. 遍历顺序不同。
2. 使用的数据结构不同。
3. BFS 更适合最短步数类问题。
4. DFS 更适合路径枚举、联通块搜索、回溯类问题。

代表题型：

1. 岛屿数量。
2. 二叉树层序遍历。
3. 单词搜索。
4. 腐烂的橘子。
5. 打开转盘锁。

### 说说你对动态规划的理解？和分治、贪心有什么区别？

动态规划适用于具有“重叠子问题”和“最优子结构”的问题。

核心思路通常是：

1. 定义状态。
2. 写出状态转移方程。
3. 明确初始化。
4. 确定遍历顺序。

和分治的区别：

1. 分治更强调子问题独立。
2. 动态规划强调子问题会重复，适合记忆化或表格化复用结果。

和贪心的区别：

1. 贪心只看当前局部最优。
2. 动态规划会系统比较多个子状态后求整体最优。

代表题型：

1. 爬楼梯。
2. 打家劫舍。

### 面试刷算法时，应该怎么组织自己的复习路径？

比较实用的复习路径是：

1. 先补复杂度、数据结构基础。
2. 再刷数组、字符串、哈希、链表这些高频基础题。
3. 然后补栈、队列、树、堆、二分、双指针、滑动窗口。
4. 再进入 DFS/BFS、回溯、贪心、动态规划。
5. 最后再看图、并查集、位运算等进阶专题。

面试准备时不要只记答案，更重要的是形成统一模板：

1. 先识别题型。
2. 再匹配数据结构。
3. 最后套解题思路和复杂度分析。

### 如果按 4 周刷题，建议的路线是什么？

一个比较稳妥的 4 周路线可以是：

1. 第 1 周：数组、字符串、哈希、双指针。
2. 第 2 周：链表、栈、队列、二分、滑动窗口。
3. 第 3 周：树、堆、DFS、BFS、回溯。
4. 第 4 周：贪心、动态规划、图、并查集，并回顾错题。

每个专题建议做三件事：

1. 先学模板。
2. 再刷代表题型。
3. 最后总结自己的错题和边界条件。

## 附：常见代码实现

### 1. 二分查找

```javascript
function binarySearch(nums, target) {
  let left = 0
  let right = nums.length - 1
  while (left <= right) {
    const mid = left + ((right - left) >> 1)
    if (nums[mid] === target) return mid
    if (nums[mid] < target) left = mid + 1
    else right = mid - 1
  }
  return -1
}
```

### 2. 反转链表

```javascript
function reverseList(head) {
  let prev = null
  let cur = head
  while (cur) {
    const next = cur.next
    cur.next = prev
    prev = cur
    cur = next
  }
  return prev
}
```

### 3. DFS 模板

```javascript
function dfs(node) {
  if (!node) return
  console.log(node.val)
  dfs(node.left)
  dfs(node.right)
}
```

### 4. BFS 模板

```javascript
function bfs(root) {
  if (!root) return []
  const queue = [root]
  while (queue.length) {
    const node = queue.shift()
    console.log(node.val)
    if (node.left) queue.push(node.left)
    if (node.right) queue.push(node.right)
  }
}
```
