## 一、算法基础认知

### 说说你对算法的理解？应用场景是什么？

算法就是“解决某一类问题的步骤和策略”。

算法题不只是写出代码，还会考察：

1. 如何抽象问题。
2. 如何选择合适的数据结构。
3. 如何控制时间和空间复杂度。
4. 如何把通用解法迁移到业务场景。

常用场景：

1. 搜索和排序。
2. 路径规划和推荐。
3. 缓存淘汰策略。
4. 文本匹配和数据去重。
5. 调度、分页、合并、统计等工程问题。

简化理解：算法负责“怎么解”，数据结构负责“怎么存”，两者通常一起出现。

### 说说你对算法中时间复杂度、空间复杂度的理解？如何计算？

时间复杂度用来描述算法执行时间随输入规模增长的趋势，空间复杂度用来描述算法额外占用空间随输入规模增长的趋势。

常见复杂度从低到高大致是：

1. `O(1)`
2. `O(log n)`
3. `O(n)`
4. `O(n log n)`
5. `O(n^2)`
6. `O(2^n)`
7. `O(n!)`

计算时通常看：

1. 循环嵌套层数。
2. 递归树展开规模。
3. 是否使用额外数组、哈希表、递归栈。

复杂度关注的是趋势和数量级，不是某次具体运行时间。

### 说说你对数据结构的理解？有哪些？区别是什么？

数据结构是组织和存储数据的方式，决定了数据的访问、插入、删除和查找效率。

常见数据结构有：

1. 数组
2. 链表
3. 栈
4. 队列
5. 哈希表
6. 树
7. 堆
8. 图
9. 集合
10. 并查集

它们的区别主要看：

1. 存储方式不同。
2. 访问方式不同。
3. 查找、插入、删除成本不同。
4. 适合解决的问题类型不同。

### 算法刷题前应该先建立哪些基础？

刷题前最先建立的是“题型识别能力”，而不是背答案。

先补这些基础：

1. 复杂度分析。
2. 常见数据结构操作。
3. 递归与迭代转换。
4. 模板化思维，比如二分、双指针、滑动窗口、DFS/BFS。
5. 边界条件意识，比如空数组、单节点、越界、重复元素。

典型题：

1. 复杂度判断题。
2. 数组遍历与原地修改。
3. 递归树基础。
4. 简单查找、插入、删除模拟题。

### 为什么递归是算法基础能力？

递归是算法题里的基础能力，因为不少数据结构天然就是“分层嵌套”的：

1. 二叉树天然递归定义
2. 分治问题天然递归拆分
3. 回溯本质是带状态恢复的递归
4. DFS 也常常直接写成递归

理解递归时需要重点关注的不是背模板，而是建立这三个意识：

1. 当前层做什么
2. 下一层交给谁
3. 什么时候停

递归题常见的错误也集中在这三点：

1. 终止条件不清楚
2. 当前层逻辑和递归逻辑混在一起
3. 返回值语义不稳定

所以不少算法课程都会先用二叉树遍历训练递归直觉，因为它结构清晰、状态简单，而且足够典型。

## 二、线性结构专题

### 说说你对数组的理解？常见的算法题思路有哪些？

数组是连续内存空间的数据结构，随机访问效率高，但中间插入和删除成本较高。

数组类题目的高频思路：

1. 遍历模拟。
2. 双指针。
3. 滑动窗口。
4. 前缀和。
5. 二分查找。
6. 原地交换。

数组题常见考察的是：“有序、连续、下标、区间”这些特征能否被利用起来。

典型题：

1. 两数之和。
2. 移除元素。
3. 合并区间。
4. 旋转数组。
5. 最大子数组和。
6. 缺失的第一个正数。

### 说说你对字符串的理解？常见的算法题思路有哪些？

字符串可以看成字符数组，所以不少字符串题最后还是数组题。

常见思路：

1. 双指针。
2. 滑动窗口。
3. 哈希计数。
4. 子串匹配。
5. 回文判断。
6. 模拟和状态机。

字符串题的关键通常是：

1. 明确子串、子序列、前缀、后缀的区别。
2. 注意大小写、空格、编码边界和下标处理。

典型题：

1. 最长公共前缀。
2. 最长无重复子串。
3. 有效字母异位词。
4. 最长回文子串。
5. 字符串相加。
6. 字符串解码。

### 说说你对链表的理解？常见的操作和题型有哪些？

链表是通过指针把节点串联起来的线性结构，不要求连续内存。

好处：插入删除方便。
问题：随机访问能力差。

链表高频题型：

1. 反转链表。
2. 快慢指针找中点或环。
3. 合并两个有序链表。
4. 删除倒数第 N 个节点。
5. 判断链表是否有环。
6. 链表分割、重排、相交判断。

这里经常考的不是 API，而是指针操作稳不稳、边界处理干不干净。

典型题：

1. 反转链表。
2. 合并两个有序链表。
3. 环形链表。
4. 删除链表倒数第 N 个节点。
5. 两两交换链表节点。
6. K 个一组翻转链表。

### 快慢指针与多指针为什么是链表题常用技巧？

链表题难的地方在于：

1. 不能随机访问
2. 不少信息只能通过“走几步”才能知道
3. 指针一旦断开，原结构就可能丢失

快慢指针之所以高频，是因为它能在一次遍历里拿到不少结构信息：

1. 找中点
2. 判断是否有环
3. 找环入口
4. 找倒数第 N 个节点

常见模型：

1. 快慢指针
   - 一个走一步，一个走两步
   - 常用于中点、环检测

2. 前后双指针
   - 一个先走 N 步，另一个再一起走
   - 常用于删除倒数第 N 个节点

3. 多指针协作
   - `prev`、`cur`、`next`
   - 常用于反转、分组翻转、局部重排

不少链表题表面不同，本质都在考：

1. 指针移动顺序
2. 边界节点处理
3. 是否会把链断开

### 环形链表专题为什么经常单独考？

环形链表题经常单独出，是因为它同时考察：

1. 快慢指针原理是否真的理解
2. 数学关系能否推导清楚
3. 指针边界是否稳定

最经典的两问：

1. 是否有环
2. 环的入口在哪里

为什么快慢指针能判断有环？

1. 如果有环，快指针一定会在环里追上慢指针
2. 如果无环，快指针会先走到 `null`

为什么相遇后再从头走能找到入口？

因为从链表头到环入口的距离、相遇点到环入口的距离，满足固定的数学关系，所以让一个指针回到头节点，另一个留在相遇点，两者都每次走一步，最终会在入口相遇。

### 说说你对栈、队列的理解？应用场景是什么？

栈是后进先出，队列是先进先出。

栈常用场景：

1. 括号匹配。
2. 单调栈。
3. 函数调用栈。
4. 表达式求值。

队列常用场景：

1. BFS 广度优先遍历。
2. 任务调度。
3. 消息缓冲。
4. 单调队列。

简化理解：栈更适合处理“最近相关”，队列更适合处理“先来先服务”。

典型题：

1. 有效括号。
2. 最小栈。
3. 用栈实现队列。
4. 用队列实现栈。
5. 单调栈求下一个更大元素。
6. 滑动窗口最大值。

### 说说你对集合和哈希表的理解？常见的操作和应用场景有哪些？

集合和哈希表背后都在用哈希思想做高效查找。

1. `Set` 更关注“元素是否存在”。
2. `Map` / 哈希表更关注“键值映射”。

常用操作：

1. 插入
2. 删除
3. 查找
4. 去重
5. 计数

常用场景：

1. 两数之和。
2. 频次统计。
3. 判断重复元素。
4. 缓存设计。
5. 字符串异位词判断。

哈希类题的核心是“用空间换时间”。

典型题：

1. 两数之和。
2. 存在重复元素。
3. 字母异位词分组。
4. 最长连续序列。
5. LRU 缓存。
6. 复制带随机指针的链表。

## 三、树、堆与图

### 说说你对树的理解？相关的操作和题型有哪些？

树是一种层次结构数据模型，常见考的是二叉树。

树的高频操作：

1. 前序、中序、后序遍历。
2. 层序遍历。
3. 求高度、深度、节点数。
4. 判断平衡、对称、是否为搜索树。
5. 最近公共祖先。

树题的常见思路：

1. 递归。
2. 迭代模拟栈。
3. BFS 队列遍历。

典型题：

1. 二叉树前中后序遍历。
2. 二叉树层序遍历。
3. 对称二叉树。
4. 验证二叉搜索树。
5. 二叉树最近公共祖先。
6. 路径总和。

### 二叉树递归遍历为什么是树题的起点？

树题里最基础、也是需要重点关注的能力，就是递归遍历。

原因是：

1. 树的定义本身就是递归定义
2. 大多数树题都能拆成“当前节点 + 左子树 + 右子树”
3. 只要把遍历框架写稳，不少树题都能基于这个思路变形

前序、中序、后序的差异，本质不是顺序死记，而是“当前节点逻辑放在哪一段执行”：

1. 前序：先处理当前节点，再递归左右子树
2. 中序：先左，再当前，再右
3. 后序：先左右子树，再处理当前节点

为什么这件事重要？

因为不同题目需要的信息时机不同：

1. 要复制树结构，常用前序
2. 二叉搜索树校验常和中序有序性相关
3. 统计高度、判断平衡，常用后序先拿子树结果再算当前节点

### 什么是二叉搜索树（BST）？为什么它经常单独考？

二叉搜索树的核心性质是：

1. 左子树所有节点值都小于根节点
2. 右子树所有节点值都大于根节点
3. 左右子树也分别是 BST

它经常单独考，是因为这个结构把“查找”这件事做得很高效。

利用 BST 的性质，不少操作都能更快：

1. 查找
2. 插入
3. 删除
4. 范围查询

典型题型：

1. 验证 BST
2. BST 第 k 小元素
3. 在 BST 中搜索节点
4. 将有序数组转成 BST

### 什么是平衡二叉树？为什么它重要？

平衡二叉树强调的是：左右子树高度差不能过大。

常见定义是：

1. 任一节点左右子树高度差不超过 1

它为什么重要？

因为普通二叉搜索树在极端情况下可能退化成链表，这样查找效率会从 `O(log n)` 退化到 `O(n)`。

平衡树的意义就是尽量让树别“长歪”。

关于平衡树，通常不会要求手写 AVL/红黑树，更常见的是：

1. 判断一棵树是否平衡
2. 求树高
3. 理解为什么平衡能保证效率更稳定

### 说说你对堆的理解？如何实现？应用场景是什么？

堆是一种特殊的完全二叉树，常见分为大顶堆和小顶堆。

特点大概是：

1. 堆顶元素最大或最小。
2. 常用于快速获取最值。

常用场景：

1. Top K 问题。
2. 优先队列。
3. 合并有序数据流。
4. 定时任务调度。
5. 堆排序。

实现上通常基于数组，通过下标关系维护父子节点。

典型题：

1. 数组中的第 K 个最大元素。
2. 前 K 个高频元素。
3. 合并 K 个有序链表。
4. 数据流中的中位数。
5. 最小堆维护 Top K。

### 堆为什么经常和 TopK、优先级问题一起出现？

堆最核心的价值，不是“它是一棵完全二叉树”，而是：

1. 能快速拿到当前最大值或最小值
2. 插入和弹出代价相对稳定

这正好适合解决两类问题：

1. **TopK**
   - 只关心前 K 大 / 前 K 小，不需要完整排序

2. **优先级调度**
   - 每次都要拿当前优先级最高或最低的元素

为什么不少 TopK 题优先考虑堆，而不是排序？

因为完整排序通常是 `O(n log n)`，而维护一个大小为 `k` 的堆，不少时候只需要 `O(n log k)`，当 `k` 明显小于 `n` 时更划算。

### 堆排序的核心思路是什么？和快速排序有什么区别？

堆排序的核心思路是：

1. 先把数组建成大顶堆
2. 每次把堆顶最大值交换到数组末尾
3. 再调整剩余部分继续保持堆结构

它的特点大概是：

1. 时间复杂度稳定在 `O(n log n)`
2. 原地排序
3. 不依赖递归划分区间

和快速排序相比：

1. 快排平均性能通常很好，但最坏可能退化到 `O(n^2)`
2. 堆排序最坏复杂度也能维持 `O(n log n)`
3. 快排通常缓存友好，工程上常更快
4. 堆排序更像“稳定保底型选手”

### 说说你对图的理解？相关操作和题型有哪些？

图用来描述点和点之间的关系，比树更通用。

常见表示方式：

1. 邻接表。
2. 邻接矩阵。

高频操作和题型：

1. DFS 深度优先遍历。
2. BFS 广度优先遍历。
3. 最短路径。
4. 拓扑排序。
5. 联通分量。
6. 并查集判连通。

图题常见于依赖关系、路径搜索、社交网络、课程安排等问题。

典型题：

1. 岛屿数量。
2. 课程表。
3. 克隆图。
4. 腐烂的橘子。
5. 单词接龙。
6. 最短路径基础题。

### 说说你对并查集的理解？应用场景是什么？

并查集是一种用于处理“元素是否属于同一集合”的数据结构。

核心操作：

1. `find`：查找根节点。
2. `union`：合并两个集合。

常用优化：

1. 路径压缩。
2. 按秩合并。

用在哪：

1. 判断连通性。
2. 岛屿数量问题。
3. 图中环检测。
4. 动态合并集合问题。

典型题：

1. 冗余连接。
2. 省份数量。
3. 岛屿数量扩展题。
4. 账号合并。

## 四、查找与排序专题

### 说说常见的排序算法有哪些？区别是什么？

常见排序算法包括：

1. 冒泡排序。
2. 选择排序。
3. 插入排序。
4. 归并排序。
5. 快速排序。
6. 堆排序。

比较时通常看这些点：

1. 时间复杂度。
2. 空间复杂度。
3. 是否稳定。
4. 是否适合大规模数据。
5. 是否适合近乎有序的数据。

典型题：

1. 数组排序。
2. 颜色分类。
3. 合并区间。
4. 会议室安排。
5. 归并排序统计逆序对。

### 说说你对冒泡排序的理解？如何实现？应用场景是什么？

冒泡排序通过相邻元素两两比较，把较大元素逐步“冒泡”到后面。

特点大概是：

1. 思路简单。
2. 平均复杂度 `O(n^2)`。
3. 稳定排序。

它更适合教学和小规模数据，不适合实际大规模场景。

典型题：

1. 基础排序模板题。
2. 近乎有序数组的教学示例。

### 说说你对选择排序的理解？如何实现？应用场景是什么？

选择排序每一轮从未排序区间中选出最小值，放到已排序区间末尾。

特点大概是：

1. 实现简单。
2. 平均复杂度 `O(n^2)`。
3. 一般不是稳定排序。

它同样更适合教学，不适合大数据量场景。

典型题：

1. 基础排序模板题。
2. 交换次数受限的教学场景。

### 说说你对插入排序的理解？如何实现？应用场景是什么？

插入排序的思路是把当前元素插入到前面已排序序列中的正确位置。

特点大概是：

1. 平均复杂度 `O(n^2)`。
2. 稳定排序。
3. 对近乎有序的数据表现较好。

实际工程里，小数组排序或某些优化场景会用到插入排序作为基础策略。

典型题：

1. 小规模数组排序。
2. 近乎有序数组排序。

### 说说你对归并排序的理解？如何实现？应用场景是什么？

归并排序采用分治思想：

1. 先把数组不断拆分。
2. 再把两个有序子数组合并。

特点大概是：

1. 时间复杂度稳定在 `O(n log n)`。
2. 稳定排序。
3. 需要额外空间。

适合用在：

1. 对稳定性有要求。
2. 链表排序。
3. 外部排序。

典型题：

1. 排序链表。
2. 合并两个有序数组/链表。
3. 逆序对统计。

### 说说你对快速排序的理解？如何实现？应用场景是什么？

快速排序的思路是：

1. 选一个基准值。
2. 把数组分成小于基准和大于基准两部分。
3. 递归处理左右区间。

特点大概是：

1. 平均复杂度 `O(n log n)`。
2. 原地排序能力较强。
3. 最坏情况会退化到 `O(n^2)`。

它是实际工程里很常见的高效排序思想。

典型题：

1. 快速排序模板。
2. 数组第 K 大元素的快速选择。
3. 颜色分类的原地分区思想。

### 说说你对二分查找的理解？如何实现？应用场景是什么？

二分查找适用于“有序区间查找”。

思路是每次比较中间值，把问题规模缩小一半。

特点大概是：

1. 时间复杂度 `O(log n)`。
2. 要求数据有序。

常用场景：

1. 查找目标值。
2. 查找左边界、右边界。
3. 答案区间二分。
4. 旋转数组、峰值、最小值类题目。

典型题：

1. 二分查找。
2. 搜索插入位置。
3. 在排序数组中查找元素的第一个和最后一个位置。
4. 搜索旋转排序数组。
5. 寻找峰值。

## 五、常见算法思想

### 说说你对双指针的理解？应用场景是什么？

双指针是一类很高频的算法技巧，通常通过两个指针在数组、链表、字符串上协同移动来优化问题。

适合用在：

1. 有序数组求和。
2. 去重。
3. 合并区间。
4. 回文判断。
5. 快慢指针处理链表问题。

它的价值是把原本 `O(n^2)` 的部分问题优化到 `O(n)`。

典型题：

1. 两数之和 II。
2. 三数之和。
3. 盛最多水的容器。
4. 移动零。
5. 判断回文串。

### 说说你对滑动窗口的理解？应用场景是什么？

滑动窗口可以看作维护一个动态区间，左右边界一边移动，一边更新窗口内状态。

适合用在：

1. 最长不重复子串。
2. 最小覆盖子串。
3. 固定长度子数组统计。
4. 连续区间和问题。

滑动窗口的关键是：

1. 窗口何时扩张。
2. 何时收缩。
3. 窗口内状态如何维护。

典型题：

1. 长度最小的子数组。
2. 最长无重复子串。
3. 最小覆盖子串。
4. 找到字符串中所有字母异位词。

### 说说你对前缀和的理解？应用场景是什么？

前缀和用于快速计算区间和。

思路：

1. 预处理出从起点到当前位置的累积和。
2. 任意区间和可用两个前缀和相减得到。

适合用在：

1. 子数组和。
2. 矩阵区域和。
3. 统计区间类问题。

它的本质还是“预处理换取快速查询”。

典型题：

1. 区域和检索。
2. 和为 K 的子数组。
3. 乘积小于 K 的子数组变体。

### 说说你对分而治之的理解？应用场景是什么？

分治思想指的是把大问题拆成若干规模更小、结构相同的子问题，分别解决后再合并结果。

经典应用：

1. 归并排序。
2. 快速排序。
3. 二叉树递归问题。
4. 大整数运算拆分。

分治的关键不只是“拆”，还包括“子问题独立”和“结果合并”。

典型题：

1. 归并排序。
2. 快速排序。
3. 不同二叉树分治递归题。

### 说说你对贪心算法的理解？应用场景是什么？

贪心算法是在每一步都做当前局部最优的选择，希望最终得到全局最优解。

前提一般是：

1. 问题具有贪心选择性质。
2. 局部最优能导向全局最优。

适合用在：

1. 区间调度。
2. 零钱问题的特定版本。
3. 跳跃游戏。
4. 最少覆盖区间。

典型题：

1. 跳跃游戏。
2. 分发饼干。
3. 无重叠区间。
4. 买卖股票的最佳时机 II。

### 说说你对回溯算法的理解？应用场景是什么？

回溯就是一种“试错 + 撤销”的搜索方法，通常拿来枚举所有可能解或找满足条件的解。

适合用在：

1. 全排列。
2. 子集。
3. 组合问题。
4. N 皇后。
5. 路径搜索。

回溯题的关键是：

1. 路径是什么。
2. 选择列表是什么。
3. 结束条件是什么。
4. 如何剪枝。

典型题：

1. 全排列。
2. 子集。
3. 组合总和。
4. N 皇后。
5. 单词搜索。

### 说说你对深度优先搜索和广度优先搜索的理解？区别是什么？

DFS 和 BFS 都是图和树的基础搜索策略。

1. DFS 倾向“一条路走到底再回头”，常用递归或栈实现。
2. BFS 倾向“按层扩展”，常用队列实现。

主要差别：

1. 遍历顺序不同。
2. 使用的数据结构不同。
3. BFS 更适合最短步数类问题。
4. DFS 更适合路径枚举、联通块搜索、回溯类问题。

典型题：

1. 岛屿数量。
2. 二叉树层序遍历。
3. 单词搜索。
4. 腐烂的橘子。
5. 打开转盘锁。

### 说说你对动态规划的理解？和分治、贪心有什么区别？

动态规划适用于具有“重叠子问题”和“最优子结构”的问题。

通常就按这个思路做：

1. 定义状态。
2. 写出状态转移方程。
3. 明确初始化。
4. 确定遍历顺序。

和分治的区别：

1. 分治更强调子问题独立。
2. 动态规划强调子问题会重复，适合记忆化或表格化复用结果。

和贪心的区别：

1. 贪心只看当前局部最优。
2. 动态规划会系统比较多个子状态后求整体最优。

典型题：

1. 爬楼梯。
2. 打家劫舍。
3. 最长递增子序列。
4. 零钱兑换。
5. 编辑距离。
6. 最长公共子序列。

### 动态规划解题时，为什么总强调“状态、转移、边界”？

因为动态规划最难的不是写代码，而是把问题抽象成一个可递推的模型。

所谓三板斧就是：

1. **状态**
   - `dp[i]` 或 `dp[i][j]` 到底表示什么

2. **状态转移方程**
   - 当前状态由哪些更小状态推出来

3. **初始边界**
   - 最小子问题是什么，初值怎么设

为什么这三步缺一不可？

1. 状态不清楚，后面的式子就没有语义基础
2. 转移不对，整个递推链条都会错
3. 边界错了，后面全会被污染

动态规划的难点通常不在代码，而在状态定义。

### 回溯和动态规划为什么看起来相似，实际却不一样？

这两个算法思想都和“子问题”有关，所以容易混淆。

相同点：

1. 都会拆解问题
2. 都常用递归或搜索过程

不同点在于：

1. **回溯**
   - 可以看作枚举 + 剪枝
   - 会尝试所有可能路径，再回退

2. **动态规划**
   - 可以看作把重复子问题结果记下来
   - 重点是避免重复计算

所以：

1. 子问题会重复出现，通常优先考虑 DP
2. 需要列举所有方案、路径、组合，通常优先考虑回溯

## 六、算法解题方法

### 做算法题时，通常会考察哪些能力？

算法题的评价维度不只看最后是否 AC，还包括：

1. 能否快速识别题型
2. 能否选对数据结构
3. 能否说明复杂度
4. 能否处理边界条件
5. 能否把暴力解法逐步优化
6. 代码是否稳定、命名是否清楚

算法题也会考察：

1. 抽象能力
2. 优化意识
3. 表达能力

### 拿到算法题后，解题顺序怎么组织？

较稳的顺序是：

1. **先确认题意**
   - 输入输出是什么
   - 是否有空数据、重复值、负数等边界

2. **先说暴力思路**
   - 表明你能先把问题解决

3. **再说优化方向**
   - 能不能利用有序性、哈希、双指针、剪枝、记忆化

4. **给出复杂度**
   - 时间复杂度、空间复杂度

5. **最后写代码**
   - 保持命名稳定，边界条件提前想清楚

为什么这样更稳？

因为算法题不只看能否写出代码，也看拆题、沟通和边界处理过程。

### 为什么刷算法不能只背答案？

因为题目在真实场景里往往会变形。

如果只背题面和答案，通常会遇到两个问题：

1. 题目一改条件就不会了
2. 不知道为什么要用这个方法

需要积累的是：

1. 题型识别能力
2. 数据结构选择能力
3. 模板迁移能力
4. 边界条件意识

所以刷题最有价值的不是“做了多少道”，而是：

1. 这题属于什么模型
2. 为什么这个模型成立
3. 如果条件变了，模型还能不能用

### 单调栈为什么会成为高频题型？

因为它特别适合解决“下一个更大 / 更小元素”“最近比我大 / 小的位置”这种问题。

它高频的原因不在于数据结构复杂，而在于：

1. 暴力法通常是 `O(n^2)`
2. 单调栈能把不少这类题压到 `O(n)`

所以一旦题目里出现“最近”“下一个”“更大 / 更小”，就应该有意识想到单调栈。

### 拓扑排序通常在什么题型里出现？

拓扑排序常见于依赖关系题。

比如：

1. 课程表
2. 编译顺序
3. 任务调度顺序

它的本质还是：在有向无环图里，找一个满足依赖约束的线性顺序。

### 并查集为什么适合处理连通性问题？

因为它的核心操作就两个：

1. 查两个点是不是同一个集合
2. 把两个集合合并

这和“是否连通”“是否属于同一组”这种问题天然匹配，所以图中联通块、岛屿归并、冗余连接判断里经常会出现它。

### 1. 两数之和

题目：给定一个整数数组 `nums` 和一个目标值 `target`，找出和为目标值的两个下标。

做法：

1. 一边遍历
2. 一边用哈希表记录“某个值出现过的位置”
3. 当前值如果能和之前某个值凑成 target，就直接返回

```javascript
function twoSum(nums, target) {
  const map = new Map()

  for (let i = 0; i < nums.length; i++) {
    const need = target - nums[i]
    if (map.has(need)) {
      return [map.get(need), i]
    }
    map.set(nums[i], i)
  }

  return []
}
```

### 2. 最长无重复子串

题目：给定一个字符串 `s`，求不含重复字符的最长子串长度。

做法：

1. 维护一个滑动窗口
2. 用 `Set` 记录窗口内字符
3. 一旦重复，就移动左指针直到重复消失

```javascript
function lengthOfLongestSubstring(s) {
  const set = new Set()
  let left = 0
  let maxLen = 0

  for (let right = 0; right < s.length; right++) {
    while (set.has(s[right])) {
      set.delete(s[left])
      left++
    }
    set.add(s[right])
    maxLen = Math.max(maxLen, right - left + 1)
  }

  return maxLen
}
```

### 3. 合并两个有序链表

题目：将两个升序链表合并为一个新的升序链表。

做法：

1. 用哑节点简化头节点处理
2. 两个指针分别遍历两个链表
3. 每次接入较小节点

```javascript
function mergeTwoLists(list1, list2) {
  const dummy = { val: 0, next: null }
  let cur = dummy

  while (list1 && list2) {
    if (list1.val < list2.val) {
      cur.next = list1
      list1 = list1.next
    } else {
      cur.next = list2
      list2 = list2.next
    }
    cur = cur.next
  }

  cur.next = list1 || list2
  return dummy.next
}
```

### 4. 环形链表 II

题目：判断链表是否有环，如果有，返回环的入口节点。

做法：

1. 快慢指针先判断是否有环
2. 相遇后，一个指针回到头节点
3. 两个指针都每次走一步，再次相遇的位置就是入口

```javascript
function detectCycle(head) {
  let slow = head
  let fast = head

  while (fast && fast.next) {
    slow = slow.next
    fast = fast.next.next

    if (slow === fast) {
      let p1 = head
      let p2 = slow

      while (p1 !== p2) {
        p1 = p1.next
        p2 = p2.next
      }

      return p1
    }
  }

  return null
}
```

### 5. 二叉树层序遍历

题目：按层返回二叉树节点值。

做法：

1. BFS
2. 队列逐层处理
3. 每层开始前记录当前层节点数

```javascript
function levelOrder(root) {
  if (!root) return []

  const result = []
  const queue = [root]

  while (queue.length) {
    const size = queue.length
    const level = []

    for (let i = 0; i < size; i++) {
      const node = queue.shift()
      level.push(node.val)

      if (node.left) queue.push(node.left)
      if (node.right) queue.push(node.right)
    }

    result.push(level)
  }

  return result
}
```

### 6. 验证二叉搜索树

题目：判断一棵二叉树是否为合法 BST。

做法：

1. BST 的中序遍历结果必须严格递增
2. 遍历过程中记录上一个访问到的值
3. 一旦当前值不大于上一个值，就返回 false

```javascript
function isValidBST(root) {
  let prev = -Infinity

  function inorder(node) {
    if (!node) return true
    if (!inorder(node.left)) return false
    if (node.val <= prev) return false
    prev = node.val
    return inorder(node.right)
  }

  return inorder(root)
}
```

### 7. 数组中的第 K 个最大元素

题目：返回数组中第 `k` 个最大的元素。

做法：

1. 维护一个大小为 `k` 的最小堆
2. 堆顶始终是当前第 `k` 大元素
3. 遍历结束时返回堆顶

```javascript
class MinHeap {
  constructor() {
    this.data = []
  }

  size() {
    return this.data.length
  }

  peek() {
    return this.data[0]
  }

  push(val) {
    this.data.push(val)
    this.bubbleUp(this.data.length - 1)
  }

  pop() {
    if (this.data.length === 1) return this.data.pop()
    const top = this.data[0]
    this.data[0] = this.data.pop()
    this.bubbleDown(0)
    return top
  }

  bubbleUp(index) {
    while (index > 0) {
      const parent = (index - 1) >> 1
      if (this.data[parent] <= this.data[index]) break
      ;[this.data[parent], this.data[index]] = [this.data[index], this.data[parent]]
      index = parent
    }
  }

  bubbleDown(index) {
    const len = this.data.length
    while (true) {
      let smallest = index
      const left = index * 2 + 1
      const right = index * 2 + 2

      if (left < len && this.data[left] < this.data[smallest]) {
        smallest = left
      }
      if (right < len && this.data[right] < this.data[smallest]) {
        smallest = right
      }
      if (smallest === index) break

      ;[this.data[index], this.data[smallest]] = [this.data[smallest], this.data[index]]
      index = smallest
    }
  }
}

function findKthLargest(nums, k) {
  const heap = new MinHeap()

  for (const num of nums) {
    if (heap.size() < k) {
      heap.push(num)
    } else if (num > heap.peek()) {
      heap.pop()
      heap.push(num)
    }
  }

  return heap.peek()
}
```

### 8. 岛屿数量

题目：给定一个由 `'1'` 和 `'0'` 组成的二维网格，计算岛屿数量。

做法：

1. 遍历整个网格
2. 每遇到一个 `'1'`，就做一次 DFS / BFS
3. 把与它连通的陆地全部标记掉
4. 每启动一次搜索，岛屿数加一

```javascript
function numIslands(grid) {
  if (!grid.length) return 0

  const rows = grid.length
  const cols = grid[0].length
  let count = 0

  function dfs(r, c) {
    if (r < 0 || r >= rows || c < 0 || c >= cols || grid[r][c] !== '1') {
      return
    }

    grid[r][c] = '0'
    dfs(r + 1, c)
    dfs(r - 1, c)
    dfs(r, c + 1)
    dfs(r, c - 1)
  }

  for (let r = 0; r < rows; r++) {
    for (let c = 0; c < cols; c++) {
      if (grid[r][c] === '1') {
        count++
        dfs(r, c)
      }
    }
  }

  return count
}
```

### 9. 爬楼梯

题目：每次可以爬 1 或 2 级台阶，问爬到第 `n` 级有多少种方法。

做法：

1. 到第 `n` 级的方法数，只和 `n - 1`、`n - 2` 有关
2. 本质是斐波那契模型
3. 用 DP 或滚动变量都可以

```javascript
function climbStairs(n) {

  let a = 1
  let b = 2

  for (let i = 3; i <= n; i++) {
    const sum = a + b
    a = b
    b = sum
  }

  return b
}
```

### 10. 零钱兑换

题目：给定不同面额硬币和总金额 `amount`，求最少需要多少枚硬币。

核心思路：

1. `dp[i]` 表示凑出金额 `i` 的最少硬币数
2. 枚举每种硬币，尝试更新 `dp[i]`
3. 如果最后仍然无法更新，就返回 `-1`

```javascript
function coinChange(coins, amount) {
  const dp = new Array(amount + 1).fill(Infinity)
  dp[0] = 0

  for (let i = 1; i <= amount; i++) {
    for (const coin of coins) {
      if (i >= coin) {
        dp[i] = Math.min(dp[i], dp[i - coin] + 1)
      }
    }
  }

  return dp[amount] === Infinity ? -1 : dp[amount]
}
```

```javascript
function permute(nums) {

  const used = new Array(nums.length).fill(false)
  const path = []

  function backtrack() {
    if (path.length === nums.length) {
      res.push(path.slice())
      return
    }

    for (let i = 0; i < nums.length; i++) {
      if (used[i]) continue
      used[i] = true

  return res
```

### 23. 课程表

```javascript
function canFinish(numCourses, prerequisites) {
  const graph = Array.from({ length: numCourses }, () => [])
  const indegree = new Array(numCourses).fill(0)

  for (const [a, b] of prerequisites) {
    graph[b].push(a)
    indegree[a]++
  }

  const queue = []
  for (let i = 0; i < numCourses; i++) {
    if (indegree[i] === 0) queue.push(i)
  }

  let count = 0
  while (queue.length) {
    const cur = queue.shift()
    count++

    for (const next of graph[cur]) {
      indegree[next]--
      if (indegree[next] === 0) queue.push(next)
    }
  }

  return count === numCourses
}
```

### 28. 合并 K 个有序链表

题目：合并多个有序链表并返回合并后的链表。

做法：用分治，两两合并。

```javascript
function mergeKLists(lists) {
  if (!lists.length) return null

  function mergeTwo(a, b) {
    const dummy = { val: 0, next: null }
    let cur = dummy

    while (a && b) {
      if (a.val < b.val) {
        cur.next = a
        a = a.next
      } else {
        cur.next = b
        b = b.next
      }
      cur = cur.next
    }

    cur.next = a || b
    return dummy.next
  }

  function divide(left, right) {
    if (left === right) return lists[left]
    const mid = left + ((right - left) >> 1)
    const l1 = divide(left, mid)
    const l2 = divide(mid + 1, right)
    return mergeTwo(l1, l2)
  }

  return divide(0, lists.length - 1)
}
```
